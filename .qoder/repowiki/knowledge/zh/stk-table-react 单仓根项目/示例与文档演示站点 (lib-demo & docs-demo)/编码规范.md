- 每个示例目录以 `index.tsx` 作为默认导出入口，并在同目录下放置类型定义 (`types.ts`)、数据源或子组件，保持单功能一目录。
- 国际化统一通过 `hooks/useI18n/index.ts` 导出的 `t()` 函数访问，键名集中在 `en.ts / zh.ts / ja.ts / ko.ts` 中维护，避免在示例内硬编码文案。
- 需要与 VitePress 主题联动的组件一律通过 `docs-demo/StkTable.tsx` 包装后的 `StkTable` 导入，而非直接引用 `src/StkTable`，以保证 theme 与 `vp-raw` 类名一致。
- 复杂示例（如 HugeData）使用 `useRef` 保存回调与最新 state 的镜像值，再在定时器/事件回调中读取 ref，规避闭包过期问题。
