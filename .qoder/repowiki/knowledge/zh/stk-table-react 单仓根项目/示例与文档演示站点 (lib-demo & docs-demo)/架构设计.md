本模块由两个并列的演示应用组成，均不发布、仅用于开发验证与文档展示。

- `lib-demo/`：最小化的 Vite + React 应用（`vite.config.ts` 通过 `server.fs.allow=['..']` 放行父目录），在 `main.tsx` 中直接 `import { StkTable } from '../lib/stk-table-react.js'` 并引入 `../lib/style.css`，用于验证构建产物可被外部项目正常消费。
- `docs-demo/`：VitePress 文档站点的 React 示例集合。入口 `StkTable.tsx` 是一个薄包装组件，将 `src/StkTable` 暴露给 VitePress 渲染环境，并通过 `hooks/useVitepress.ts` 中的 `useVitepressData` hook 读取 `documentElement` 上的 `dark` class 与 `lang`，用 `MutationObserver` 同步 VitePress 主题与语言状态到 React 子树。
- 示例按功能域分层组织：`basic/` 覆盖基础能力（对齐、固定列、排序、树形等），`advanced/` 聚焦高级特性（自定义单元格、区域选择、行拖拽、高亮、虚拟滚动等），`demos/` 是集成型复杂场景（HugeData、Matrix、PanelTree、VirtualList 等），`components/` 与 `hooks/` 为跨示例复用的 UI 控件与 i18n 钩子。
- 依赖方向单向：所有示例 → `docs-demo/StkTable.tsx` → `src/StkTable`；`lib-demo` 则绕过源码直接消费 `lib/` 产物，形成两条独立的消费路径。
